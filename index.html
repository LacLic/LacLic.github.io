Codeforces-Round726(Div.2)D - Deleting Divisors

Posted on 2021-07-03

题意

Alice和Bob在玩一个游戏：给定一个数n，从Alice开始，每个人进行一次操作：将n减去一个n的约数（不能是1或n），然后生成的数交给另一个人

若轮到这个人的时候，没有办法进行任何操作时，对方就赢了，输出获胜人的名字

思路

根据题意和样例解释，这是给定输入后，结果就已经确定的问题

考虑必胜情况：

  1. n为质数，Alice必输
  2. 若拿到了奇数，该人必输
  3. 若拿到了非$2^k$的偶数，该人必赢
  4. 拿到了$2^k$的偶数，若k为偶数，该人必赢，若k为奇数，该人必输

情况1极其显然

情况2和情况3证明如下：

若拿到了奇数，则该奇数只能减去一个奇数因子，还给对方一个偶数

由于约数的特性：若 $n=m\cdot p$ (其中m为奇数)，则$n-m=m\cdot(p-1)$也就是说，奇数$m$仍然为$n$的约数，不会跳到情况4

若拿到了非$2^k$的偶数，必定含一个奇数因子，此时，可以丢给对方一个奇数

因此，拿到这样的偶数的人总有办法丢一个奇数给对方，而拿到的奇数有可能为质数，所以拿到这样偶数的人必赢

情况4：

若拿到了$2^k$的偶数，则有2种选择（有一种可以舍去）

  1. $n = 2^k-2^{k-1} = 2^{k-1}$ 分解为子问题：另一个人拿到了$2^{k-1}$
  2. $n$减去一个其他偶数，导致 $n = 2^k - 2^m = 2^m\cdot (2^{k-m}-1)$，其中$2^{k-m}-1$必为奇数，则对方拿到了一个非$2^k$的偶数，回到了情况3，对方必胜，不合题意，舍去

故只有选择1成立

接下来单独讨论一下选择1就好了：为2的时候Bob赢；为4的时候可以转换成对方拿到2，Alice赢；为8的时候，可以转换为对方拿到4，Bob赢……如此交替…

Codeforces-Round726(Div.2)E - Erase and Extend

Posted on 2021-07-03

题意

给定一个n和n长度的字符串s以及一个值k，要求你取s的子串（必须从首字符开始，且至少有1个字符），然后不断复读s()直至s的长度大于等于k，取前k个字符为新串并输出

要求你根据给定的字符串复读出一个字典序最小的新串

思路

首先，最简单的情况（记A），dcba这种字典序递减的字符，遇到字典序变小时必定取该字符串

其次，若碰到dcbe，这种e比首字符d字典序靠后的时候，必定不如直接复读dcb，直接break后输出

最后也是难点，考虑新串是否比复读更优即可

对于已经准备好的原串（dcb），我们比较后续串是否比复读原串更优：比对后续串和原串中的每个元素，如果一样（dcbdc），则继续比对；如果后续串更优（dcbdca），说明加上后续串（dca）后，结果更优；如果后续串不优，则break跑路

如果超出原串大小怎么办？那说明后续串和复读原串效果一样（必然是不断dcbdcbdcb），可以继续比对，不用在意

Codeforces-Round726(Div.2)F - Figure Fixing

Posted on 2021-07-03

题意

给定一个n节点，m条边的无向图，每个节点都有一个init初始值和一个dest目标值，保证给定的图是一个连通图

现在，你可以对每条边进行操作：让这条边的两个端点节点的当前值同时加一个值x（可为负）

现在，请问这张图是否能让每个节点达到dest目标值

思路

思路A（官方题解）

首先，我们不用在乎初始值，只要计算与目标值的差值即可

由于操作的性质，同一边上的两个节点的值可以相消

故我们采取奇偶染色法，颜色A的值相加，颜色B的值相加

若出现同一个节点既可以是A也可以是B（称为特殊节点），则说明它可以不断产生偶数

那么把颜色A节点差值取和，B同理，若A==B，则YES，若A-B为奇数（!=0）则NO

如果出现特殊节点，则A-B为偶数时YES；如果没有特殊节点，则A-B为偶数时NO

思路B（我的思路）

首先，我们不用在乎初始值，只要计算与目标值的差值即可

其次，经过有限次操作可以把所有值集中到某一个节点去

如果集中后已满足结果，则YES

如果集中后该节点为剩余值为奇数，则NO

另外，由于环形的出现，奇数节点的环会成为一个“偶数发生器”

例：a、b、c相连：a和b同加1，b和c同减1，c和a同加1，最终，a凭空+2，同理b或者c也行

所以，如果存在这样一个环，则剩余值为偶数也可以YES；如果不存在，剩余值为偶数依旧NO

判断这种环的存在性，我们可以用奇偶染色法

  • 0 如果
  • -1 如果

思路

二进制操作问题，我们全程采取二进制进行思考

Codeforces-Round727(Div.2)F - Strange Array

Posted on 2021-07-03

线段树真的是太好用了。。。

题意

给定一个长度为n的数组()，请你找出每个元素的奇怪度(strange)

定义奇怪度：对于每个元素，选定()，将区间的元素进行从小到大排序，，其中median表示中间数（若为偶数则取中间靠大的那个）

思路

显然，对于每一个元素，其奇怪值只和左边（小于）元素个数、右边（大于）元素个数、相等元素个数有关，而且，左边元素个数可以和右边元素个数相抵消

那么，我们取大于它的元素为1，小于它的元素为-1

等于它的元素，视情况讨论，因为该元素可以在左边（此时，把尽可能多的相等元素丢到右边去，来拉开和中间数的距离），或者在右边（同理），两种情况都算一下，取大就好

由于和大小有关，方便起见，我们按从小到大的顺序（而不是原始位置），对每一个元素进行探讨

探讨时，我们先将假定改元素在左边（等于它的元素都置为1），计算得出结果，再假定该元素在右边，此时，我们把所有其他等于它的元素都置为-1，计算得出结果后，取大值。然后再探讨更大的元素

使用线段树维护这些元素：在查询的时候，每一层都根据下一层左右大小个数最多的元素，来维护最值（详情见代码）
